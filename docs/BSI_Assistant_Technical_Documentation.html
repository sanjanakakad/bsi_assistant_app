3. Chat Mode — Backend Steps (Phases 1–3)

  1. User input via st.chat_input
  2. split_components() — parse delimiters
  3. map_to_ind() — regex + synonym lookup + fuzzy match (≥86%)
  4. OpenAI thread created; Assistant run with file search
  5. Response with threat titles and PDF citations rendered in chat

4. Matrix Mode — Backend Steps (Phases 4–5)

  1. Architecture selected from component_compositions.yaml
  2. aggregate_threats() loads threats from JSON per BSI module
  3. VDI C/I/A/S flags applied from YAML mapping (deterministic)
  4. Expert templates merged; matrix shown in st.data_editor
  5. Audit labels: 🟢 BSI | 🟡 Modified | 🔵 Expert → CSV export

5. Design Principles

  • Hybrid AI: LLM for reading; rules for compliance outputs
  • Auditability: Origin and modification status on every row
  • Anti-hallucination: VDI flags are human-reviewed YAML, not LLM-generated
